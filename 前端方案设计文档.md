# AI+政务问答 H5 前端方案设计文档

## 一、项目概述

### 1.1 项目背景
本项目是黄山市人民政府 AI+政务问答系统的前端 H5 页面，基于 Vue 3 + Vite 技术栈开发，对接后端的 RAG (Retrieval-Augmented Generation) 问答服务。

### 1.2 核心功能
- AI 智能问答对话
- 历史对话管理
- 高频办事快捷入口
- 移动端适配

---

## 二、技术架构

### 2.1 技术栈选型

```
┌─────────────────────────────────────────────────────────────────┐
│                        技术栈架构                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  框架层:    Vue 3 (Composition API) + Pinia (状态管理)           │
│  构建工具:  Vite 5 (快速构建、热更新)                            │
│  样式方案:  SCSS (CSS预处理器)                                   │
│  网络请求:  Axios (HTTP客户端)                                   │
│  文本渲染:  Marked + Highlight.js (Markdown渲染)                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 选型理由

| 技术 | 选型理由 |
|------|----------|
| **Vue 3** | 组合式 API 更灵活，性能更好，TypeScript 支持更好 |
| **Vite** | 开发启动快（秒级），热更新快，构建产物优化好 |
| **Pinia** | Vue 官方推荐，TypeScript 友好，API 更简洁 |
| **SCSS** | 变量、嵌套、混入等特性，样式更易维护 |
| **Axios** | 拦截器、取消请求、自动转换 JSON，功能完善 |

### 4.3 API 接口设计

```javascript
// api/chat.js

/**
 * 发送对话消息
 * POST /chat
 * @param {number} knowledgeId - 知识库ID
 * @param {Array} messages - 消息历史
 * @returns {Promise<RAGResponse>}
 */
export const sendChatMessage = async (knowledgeId, messages) => {
  const response = await apiClient.post('/chat', {
    knowledge_id: knowledgeId,
    message: messages
  })
  return response
}

// 请求体示例
{
  "knowledge_id": 1,
  "message": [
    { "role": "user", "content": "如何办理养老保险？" }
  ]
}

// 响应体示例
{
  "request_id": "uuid",
  "message": [
    { "role": "user", "content": "如何办理养老保险？" },
    { "role": "system", "content": "办理养老保险需要..." }
  ],
  "response_code": 200,
  "response_msg": "ok",
  "process_status": "completed",
  "processing_time": 1.23
}
```

### 5.2 历史对话管理

```javascript
// 使用 localStorage 持久化存储

// 保存对话列表
const saveConversations = (conversations) => {
  localStorage.setItem('conversations', JSON.stringify(conversations))
}

// 加载对话列表
const loadConversations = () => {
  const saved = localStorage.getItem('conversations')
  return saved ? JSON.parse(saved) : []
}

// 数据结构
{
  id: 'timestamp',
  title: '对话标题（取第一条消息前20字）',
  timestamp: 'ISO时间戳',
  messages: [
    { id, role, content, timestamp }
  ]
}
```

### 5.3 Markdown 渲染

```javascript
// utils/markdown.js
import { marked } from 'marked'
import hljs from 'highlight.js'

marked.setOptions({
  highlight: function(code, lang) {
    if (lang && hljs.getLanguage(lang)) {
      return hljs.highlight(code, { language: lang }).value
    }
    return hljs.highlightAuto(code).value
  },
  breaks: true,
  gfm: true
})

export const renderMarkdown = (markdown) => {
  return marked.parse(markdown)
}
```

## 八、开发运行指南

### 8.1 环境要求
- Node.js >= 16
- npm >= 8

### 8.2 本地开发

```bash
# 1. 进入前端目录
cd frontend

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev

# 4. 浏览器访问 http://localhost:3000
```

### 8.3 后端接口配置

```javascript
// vite.config.js 中配置代理
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8000',  // 后端服务地址
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api/, '')
    }
  }
}
```

### 8.4 生产构建

```bash
npm run build
```

构建产物在 `dist/` 目录，可直接部署到静态服务器。
