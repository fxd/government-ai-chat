| 组件 | 选择 | 理由 |

## 二、服务器环境准备

### 2.1 购买阿里云服务器

**推荐配置：**
- **类型**: 轻量应用服务器（适合个人/小项目）或 ECS
- **配置**: 1核2G 或 2核4G
- **系统**: Ubuntu 22.04 LTS
- **带宽**: 3-5Mbps
- **费用**: 约 100-300元/年

**购买步骤：**
1. 访问 [阿里云官网](https://www.aliyun.com/)
2. 选择「轻量应用服务器」或「云服务器ECS」
3. 选择地域（建议选离用户近的，如华东1杭州）
4. 选择镜像：Ubuntu 22.04
5. 购买并记录公网IP

### 2.2 域名与备案

**域名购买：**
```
1. 在阿里云购买域名（如: huangshan-ai.gov.cn）
2. 域名实名认证
3. 域名备案（必须！国内服务器必须备案）
   - 备案流程约 7-20 个工作日
   - 需要准备：身份证、营业执照（企业）、承诺书等
```

**临时方案（无域名）：**
```
使用 IP 地址访问，或配置本地 hosts 文件测试
```

## 三、服务器部署步骤

### 3.1 连接服务器

```bash
# 使用 SSH 连接服务器
ssh root@你的服务器IP

# 输入密码（购买时设置的密码）
```

### 3.2 安装必要软件

```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Nginx
apt install nginx -y

# 安装 Node.js (用于构建，可选)
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
apt install -y nodejs

# 验证安装
nginx -v
node -v
npm -v
```

### 3.3 配置 Nginx

```bash
# 创建网站目录
mkdir -p /var/www/government-ai

# 设置权限
chown -R www-data:www-data /var/www/government-ai
```

**Nginx 配置文件：**

```bash
# 编辑配置文件
vim /etc/nginx/sites-available/government-ai
```

```nginx
server {
    listen 80;
    server_name your-domain.com;  # 你的域名，或填 _ 表示任意
    
    # 日志配置
    access_log /var/log/nginx/government-ai.access.log;
    error_log /var/log/nginx/government-ai.error.log;
    
    # 前端静态资源
    location / {
        root /var/www/government-ai;
        index index.html;
        
        # 关键：支持 Vue Router 的 history 模式
        try_files $uri $uri/ /index.html;
        
        # 缓存静态资源
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }
    
    # API 代理 - 转发到后端服务
    location /api/ {
        proxy_pass http://localhost:8000/;  # 后端服务地址
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
    
    # Gzip 压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/javascript application/xml+rss application/json;
}
```

```bash
# 启用配置
ln -s /etc/nginx/sites-available/government-ai /etc/nginx/sites-enabled/

# 删除默认配置（可选）
rm /etc/nginx/sites-enabled/default

# 测试配置
nginx -t

# 重启 Nginx
systemctl restart nginx
```

### 3.4 部署前端代码

**方式一：本地构建后上传（推荐）**

```bash
# 在本地项目目录执行
npm run build

# 使用 scp 上传到服务器
scp -r dist/* root@你的服务器IP:/var/www/government-ai/
```

**方式二：服务器上构建**

```bash
# 在服务器上
mkdir -p /opt/project
cd /opt/project

# 上传源码（使用 git 或 scp）
# git clone your-repo.git

# 安装依赖并构建
cd frontend
npm install
npm run build

# 复制到网站目录
cp -r dist/* /var/www/government-ai/
```

### 3.5 配置 HTTPS (SSL证书)

**使用 Certbot 免费证书：**

```bash
# 安装 Certbot
apt install certbot python3-certbot-nginx -y

# 申请证书（需要有域名）
certbot --nginx -d your-domain.com

# 按提示操作，选择重定向 HTTP 到 HTTPS

# 自动续期测试
certbot renew --dry-run
```

**配置成功后，Nginx 会自动修改配置：**

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    
    # ... 其他配置
}

server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

## 四、后端服务部署

### 4.1 部署后端（在同一台服务器）

```bash
# 进入后端项目目录
cd /opt/project/backend

# 安装 Python 依赖
pip install -r requirements.txt

# 使用 systemd 管理服务
vim /etc/systemd/system/government-ai.service
```

```ini
[Unit]
Description=Government AI Backend
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/project/backend
Environment="PATH=/usr/local/bin"
ExecStart=/usr/local/bin/python main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

```bash
# 启动服务
systemctl daemon-reload
systemctl enable government-ai
systemctl start government-ai

# 查看状态
systemctl status government-ai

# 查看日志
journalctl -u government-ai -f
```

### 4.2 跨域配置

如果前后端分开部署，需要配置 CORS：

```python
# 后端 main.py 添加
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://your-domain.com"],  # 前端域名
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

## 五、自动化部署（CI/CD）

### 5.1 GitHub Actions 配置

```yaml
# .github/workflows/deploy.yml
name: Deploy to Aliyun

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
    
    - name: Install dependencies
      run: |
        cd frontend
        npm ci
    
    - name: Build
      run: |
        cd frontend
        npm run build
    
    - name: Deploy to Aliyun
      uses: appleboy/scp-action@master
      with:
        host: ${{ secrets.HOST }}
        username: root
        password: ${{ secrets.PASSWORD }}
        source: "frontend/dist/*"
        target: "/var/www/government-ai"
        strip_components: 2
```

### 5.2 配置 Secrets

在 GitHub 仓库设置中添加：
- `HOST`: 服务器IP
- `PASSWORD`: SSH密码（或使用 SSH_KEY）

```bash
# 备份脚本
vim /opt/backup.sh

#!/bin/bash
DATE=$(date +%Y%m%d)
BACKUP_DIR="/opt/backups/$DATE"

mkdir -p $BACKUP_DIR

# 备份前端代码
cp -r /var/www/government-ai $BACKUP_DIR/

# 备份 Nginx 配置
cp /etc/nginx/sites-available/government-ai $BACKUP_DIR/

# 备份后端代码

# 压缩
tar -czf $BACKUP_DIR.tar.gz $BACKUP_DIR
rm -rf $BACKUP_DIR

# 保留最近7天的备份
find /opt/backups -name "*.tar.gz" -mtime +7 -delete
```

```bash

crontab -e
```

## 十、快速部署脚本

```bash
#!/bin/bash
# deploy.sh - 一键部署脚本

echo "===== 开始部署 ====="

# 1. 构建前端
echo "构建前端..."
cd frontend
npm install
npm run build

# 2. 上传到服务器
echo "上传到服务器..."
read -p "请输入服务器IP: " HOST
read -p "请输入服务器密码: " PASSWORD

sshpass -p $PASSWORD scp -r dist/* root@$HOST:/var/www/government-ai/

# 3. 重启 Nginx
echo "重启 Nginx..."
sshpass -p $PASSWORD ssh root@$HOST "systemctl restart nginx"

echo "===== 部署完成 ====="
echo "访问地址: http://$HOST"
```
